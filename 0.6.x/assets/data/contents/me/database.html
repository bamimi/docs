Database

# Introduction

Almost every modern web application interacts with a database. BAMIMI interact Sequelize to support SQL database issues. So you can research deeply at Sequelize, in BAMIMI Document, we only guide how BAMIMI applies Sequelize inside

Additionally, MongoDB is supported via the Mongoose package, which is officially maintained by MongoDB.

# Configuration

The configuration for BAMIMI's database services is located in your application's configs/database.js configuration file. In this file, you may define all of your database connections, as well as specify which connection should be used by default. Most of the configuration options within this file are driven by the values of your application's environment variables. Examples for most of BAMIMI's supported database systems are provided in this file.

By default, which is a Docker configuration for developing BAMIMI applications on your local machine. However, you are free to modify your database configuration as needed for your local database.

# Init

BAMIMI is managing centrally in kernel/database with function getDB, we suggest that you should want to use through in kernel for management instance and you database easily 

//@iApp/ORMs/User.js
const db = require("@iKernel/database").get();

Another way, you can select the database which you want by transfer name of database type: 

//@iApp/ORMs/User.js
const db = require("@iKernel/database").get({ useDatabase: 'sql' });
