# The Root Directory

# The App Directory

The app directory contains the core code of your application. We'll explore this directory in more detail soon; however, almost all of the code in your application will be in this directory.

# The Bootstrap Directory

The bootstrap directory contains index.js file to bootstrap the framework. This directory also contains a cache directory containing skeletonized files for performance optimization, such as routes

# The Configs Directory

The configs directory, as its name suggests, contains all your application configuration files. You should read through all of these files and familiarize yourself with all the options available to you.

# The Database Directory

The database directory contains your database migrations and seeds. If you wish, you may also use this directory to hold an SQLite database.

# The Interfaces Directory

The interfaces directory contains interfaces for web and API, we understand that all communication between users and applications, between applications and applications, needs to go through interfaces.

# The Kernel Directory

The kernel directory contains all service declarations and instantiations, including services integrated into the main activity flow, which will be active throughout the life of the framework.

# The Libs Directory

The libs directory contains the entire library developed specifically for your project.

# The Modules Directory

The modules directory contains functional modules you can split in your project, this is just an optional.

# The Notifications Directory

The notifications directory contains extensions of information you will use in projects such as email templates.

# The Public Directory

The public directory houses your assets such as images, JavaScript, CSS and content you want to publish, photos, etc.

# The Routes Directory

The routes directory contains all of the route definitions for your application. By default, four route types are included with BAMIMI: api, cli, web, socket

The api directory contains routes that are intended to be stateless, so requests entering the application through these routes are intended to be authenticated via tokens and will not have access to session state.

The cli directory is where you may define all of your closure based console commands. Each closure is bound to a command instance allowing a simple approach to interacting with each command's IO methods. Even though this file does not define HTTP routes, it defines console based entry points (routes) into your application. You may also schedule tasks in the cli.

The web directory contains routes that BAMIMI places in the web middleware group, which provides session state, CSRF protection, and cookie encryption. If your application does not offer a stateless, RESTful API then all your routes will most likely be defined in this.

The socket directory contains all the paths for real-time processing, or publishing events to the client.

# The Storage Directory

The storage directory contains all the content you want to save such as file uploads, cache, content system.

# The Tests Directory

The tests directory contains your automated tests. Examples of jest.js tests This folder will be placed outside src, in fact during the project build process this folder will not be built

# The Node Modules Directory

The node_modules directory contains your Packages dependency

# The Commands Directory

The commands directory contains all processing logic of the custom bamimi-enjoy commands for your application. command.

# The Emails Directory

The emails directory contains contains all the functions that represent emails sent by your application. Mail objects allow you to encapsulate all the logic that builds an email in one simple class. If you want to can create these by bamimi-cli email:generate

# The Http Directory

The http directory contains your controllers, middleware, and form requests. Almost all of the logic to handle requests entering your application will be placed in this directory.

The controllers directory contains logic files manage the processing logic for HTTP requests. They act as intermediaries between requests from the client (request) and responses from the server (response).

The errors directory contains all custom http errors for your application.

The middleware directory contains files that support logic handling and handling of problems in the entire request.

The requests directory contains processing files for input requests, such as data validation, you can create it with the command bamimi-cli request:generate

# The Jobs Directory

The jobs directory contains the queueable jobs for your application. Jobs may be queued by your application or run synchronously within the current request lifecycle. Jobs that run synchronously during the current request are sometimes referred to as "commands" since they are an implementation of the command pattern. If you want to can create these by bamimi-cli jobs:generate

# The ORMs Directory

The ORMs(Object Relationship Models) directory contains all of your Database model. The ORMs included with BAMIMI kernel a beautiful, simple Sequelize.js implementation for working with your database. Each database table has a corresponding "Object Relationship Model" which is used to interact with that table. Models allow you to query for data in your tables, as well as insert new records into the table.

# The services Directory

The services directory contains business logic files
