Installation

# Creating a BAMIMI Application

# Installing Node.js and the BAMIMI Cli

Before creating your first BAMIMI application, make sure that your local machine has Node.js, NPM installed.

Once you have NodeJs and NPM on your computer, we proceed to install BAMIMI Cli: 

//with npm
npm install -g @knfs-tech/bamimi-cli
Or
//with yarn
yarn global add @knfs-tech/bamimi-cli
# Creating an Application

Once you've installed Node.js, NPM, and the BAMIMI Cli, you're ready to create a new BAMIMI application: 

//Normal
bamimi-cli create demo-app
Or
//With Npx
npx bamimi-cli create demo-app

The BAMIMI Cli will prompt you to select your preferred testing framework, database, and manager kit:

You can choose whether to use Docker to build and deploy later 

? Would you like to include a Dockerfile? (Y/n)

Do you want to use lint? This can create constraints that help you follow conventions, making your code syntax clearer and more consistent

? Would you like to include ESLint configuration? (Y/n)

TDD standards are very popular, you can also choose to use testing to increase code quality later

? Would you like to include Test configuration? (Y/n)

Choose for yourself a type of database to use, here we will demo so choose none to save time 

? Which type of database would you like to use?
  SQL
  NoSQL
> none

We will choose a tool to manage packages, as by default the local machine usually has npm so we choose npm. 

? Which type of package manager would you like to use?
  yarn
> npm
  pnpm
  none

And now we will wait for the application system to create the structure and install the libraries

# Start project in local

Once the application has been created, you can start the local development server, which uses the dev script: 

//With Npm
npm run start-dev
Or
//With Npm
yarn start-dev
